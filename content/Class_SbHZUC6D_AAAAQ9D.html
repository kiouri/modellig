Уникальный идентификатор записи помещения public string

Существует два вида индентификационных кодов – идентификационный код записи и идентифицикационный код адресного объекта (см. ниже). Идентификационный код записи используется как ключевое поле для идентификации записей по вносимым изменениям. По этому коду отслеживается вся история изменений.

Глобальный уникальный идентификатор помещения public string

Атрибут должен содержать глобальный уникальный идентификатор помещения. Существует два вида индентификационных кодов – идентификационный код записи и идентифицикационный код помещения. Идентификационный код помещения присваивается помещению при его создании и впоследствии не меняется, за исключением операции дробления. В случае дробления из одного помещения образуется несколько и у каждого помещения должен быть свой уникальный идентификационный код. Для формирования идентификационных кодов используется технология GUID (Globally Unique Identifier) – шестнадцатибайтовый (128-битный) глобальный уникальный идентификатор.

Уникальный идентификатор родительского объекта (дома) public string

Атрибут должен содержать глобальный уникальный идентификатор адресного объекта по реестру адресообразующих элементов. Содержит 36 символов.

Код региона (код субъекта Российской Федерации) public string

Требования к атрибуту не определены. Рекомендуемые требования к значениям атрибута: Атрибут должен содержать код субъекта Российской Федерации, выбранный из актуальной версии справочника «Коды субъектов Российской Федерации (регионов)». Содержит 2 символа.

Дата внесения (обновления) записи public Дата

Требования к атрибуту не определены. Рекомендуемые требования к значениям атрибута: Атрибут должен содержать дату внесения записи. Значение атрибута представлено в виде: ДД-ММ-ГГГГ, где ГГГГ – год; ММ – месяц; ДД – день.
